## Runtime Summary

- Shape: session -> starmap `10x10` -> 10 deterministic seeded starsystems, each `10x10` -> one planet
  per starsystem, each planet with a `10x10` submap.
- Logical locations: `100 + 10 * 100 + 10 * 100 = 2,100`.
- Sparse residency: only the active theater stack is resident; inert theaters/cells remain non-resident.
- Descent/ascent: deterministic starmap -> starsystem -> planet and planet -> starsystem -> starmap
  transitions are recorded as per-step residency reports.
- Value no-op / parity: materialized I8 field values are generated by a stable CPU oracle; residency
  changes where/whether data is resident, never the values.
- Replay: identical seed + access pattern yields identical residency reports and checksum.
